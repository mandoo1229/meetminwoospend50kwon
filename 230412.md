# 버전 관리 정책
1. 메인 프로젝트 생성 후 조원들이 각각 Fork한다.
2. 각자 수정사항을 Push하여 Pull request를 생성한다.
3. 회의 시 혹은 수시로 수정 사항을 확인하여 merge한다.
